# คู่มือการใช้งาน BuyDi สำหรับผู้ซื้อ

# **คู่มือการใช้งาน BuyDi สำหรับผู้ซื้อ และเอกสารอ้างอิงเชิงปฏิบัติการ (ฉบับปรับปรุง 2026-03-30)**

เอกสารฉบับนี้จัดทำขึ้นเพื่อใช้เป็นมาตรฐานในการสื่อสารกับลูกค้า (Buyer) และเป็นบรรทัดฐานสำหรับทีมกฎหมาย (Legal) และทีมปฏิบัติการ (Operations) โดยอ้างอิงตาม PRD เวอร์ชัน 2.0 (BuyDiConnect)

---

## **Part 1: เส้นทางการเดินทางของผู้ซื้อ (Buyer Journey)**

เพื่อให้สอดคล้องกับ **EPIC-005: Buyer Shopping Experience**, ขั้นตอนการใช้บริการของลูกค้ามีดังนี้:

### **1\. การเลือกชมและตัดสินใจ (Product Discovery)**

* **ช่องทาง:** ลูกค้าสามารถเข้าถึงสินค้าได้โดยตรงผ่านแพลตฟอร์ม หรือผ่านลิงก์แนะนำจาก Creator (**EPIC-004: Affiliate Tracking**)
* **มาตรฐานข้อมูล:** สินค้าทุกชิ้นได้รับการตรวจสอบ (Approve) จาก Admin เพื่อความถูกต้องของราคาและคุณภาพ
* **เครื่องมือช่วยตัดสินใจ:** ระบบแสดงผลรีวิวและคะแนน (Ratings) จากผู้ซื้อจริงเท่านั้น (**PRD 4.2.9**)

  ### **2\. การสั่งซื้อและชำระเงิน (Secure Checkout)**

* **รูปแบบธุรกรรม:** เป็นการชำระเงินรายครั้ง (Transaction-based) **ไม่มีการเก็บเงินค้างไว้ในระบบ Wallet ของผู้ซื้อ**
* **ช่องทางที่รองรับ (PRD 4.7.1):**
  1. **QR PromptPay:** ระบบสร้าง Dynamic QR Code เพื่อความสะดวกและแม่นยำ
  2. **บัตรเครดิต/เดบิต:** ผ่านระบบ Gateway มาตรฐานสากล
  3. **เก็บเงินปลายทาง (COD):** ผ่านระบบพันธมิตรไปรษณีย์ไทย
* **การยืนยัน:** ลูกค้าจะได้รับ Email ยืนยันคำสั่งซื้อทันทีเมื่อการชำระเงินเสร็จสมบูรณ์
* **การยกเลิกคำสั่งซื้อ:** ลูกค้าสามารถยกเลิกคำสั่งซื้อได้เฉพาะก่อนสถานะ Processing (ก่อนร้านค้าเริ่มแพ็กสินค้า) เท่านั้น หากไม่ชำระเงินภายในเวลาที่กำหนด ระบบจะยกเลิกคำสั่งซื้อโดยอัตโนมัติ

  ### **3\. การจัดส่งและติดตามสถานะ (Fulfillment & Tracking)**

* **การขนส่ง:** ดำเนินการโดยผู้จัดส่งมืออาชีพ (Thai Post / GetShip) มีเลขพัสดุระบุชัดเจน
* **การติดตาม:** ลูกค้าตรวจสอบสถานะแบบ Real-time ได้ผ่านเมนู "คำสั่งซื้อของฉัน" จนกระทั่งสินค้าถึงมือ (**Delivered**)

  ### **4\. การตรวจสอบสินค้าและระบบคุ้มครองผู้ซื้อ (Escrow Policy)**

* **ระยะเวลารับประกันตรวจสอบสินค้า:** ระบบคุ้มครองผู้ซื้อเป็นเวลา **3 วัน** นับจากวันที่ได้รับสินค้า
* **การยืนยัน:** เมื่อตรวจสอบสินค้าแล้วถูกต้อง ลูกค้ากดปุ่ม "ได้รับสินค้าแล้ว" เพื่อเสร็จสิ้นธุรกรรม
* **กรณีไม่กดรับสินค้า:** ระบบจะทำการยืนยันอัตโนมัติ (Auto-complete) เมื่อพ้นกำหนด 3 วัน เพื่อดำเนินการโอนเงินให้ผู้ขายตามรอบบัญชี

## **Part 2: นโยบายการคืนเงินและคืนสินค้า (Return & Refund Policy)**

เนื่องจากไม่มีระบบ Wallet สำหรับผู้ซื้อ นโยบายการคืนเงินจึงยึดหลัก **"คืนกลับสู่แหล่งจ่ายเดิม"** ดังนี้:

### **1\. เงื่อนไขการขอคืนเงิน (Return Conditions)**

* ลูกค้าต้องแจ้งความประสงค์ผ่านระบบภายใน **3 วัน** หลังได้รับสินค้า (สถานะ Delivered) พร้อมแนบหลักฐาน
* ครอบคลุมกรณี: สินค้าชำรุด, สินค้าไม่ตรงตามรายละเอียด, หรือไม่ได้รับสินค้า

### **2\. ขั้นตอนการพิจารณาคืนสินค้า (Return Process)**

เมื่อ Buyer ยื่นคำร้อง → Seller มีเวลา **3 วัน** ในการตรวจสอบและเลือก 1 ใน 3 ทางเลือก (หากไม่ตอบภายใน 3 วัน ระบบจะเลือกทางเลือก A อัตโนมัติ):

| ทางเลือก | รายละเอียด |
| ----- | ----- |
| **A: คืนสินค้า + คืนเงิน** | Buyer ส่งคืนสินค้าภายใน 3 วัน → Seller ตรวจสอบของภายใน 3 วัน (ไม่ตรวจ = auto approve) → Seller รับผิดชอบค่าส่ง 2 ขา → Admin คืนเงิน Buyer |
| **B: คืนเงินอย่างเดียว** | ไม่ต้องคืนสินค้า → Seller สละสิทธิ์รับสินค้าคืน → Admin คืนเงิน Buyer ทันที |
| **C: ไม่อนุมัติ** | Seller ใส่เหตุผล → Buyer โต้แย้งได้ภายใน 3 วัน → ไม่โต้แย้ง = ปิดเคส (Order Complete) / โต้แย้ง → เข้าสู่ Dispute |

### **3\. การระงับข้อพิพาท (Dispute — Admin ตัดสิน ผลถือเป็นที่สิ้นสุด)**

| ผลตัดสิน | สินค้าอยู่ที่ Buyer | สินค้าอยู่ที่ Seller |
| ----- | ----- | ----- |
| **Buyer Win** | คืนเงินเต็ม + Buyer เก็บสินค้า | คืนเงินเต็ม + Seller รับค่าส่ง 2 ขา |
| **Seller Win** | ไม่คืนเงิน + Buyer เก็บสินค้า | ไม่คืนเงิน + Seller รับค่าส่งขากลับ 1 ขา |

### **4\. ช่องทางการคืนเงิน (Refund Channels)**

| ช่องทางที่ชำระมา | วิธีการคืนเงิน | ระยะเวลาดำเนินการโดยประมาณ |
| ----- | ----- | ----- |
| **บัตรเครดิต/เดบิต** | คืนวงเงินเข้าบัตรเดิม (Void/Refund) | 7-15 วันทำการ (ตามนโยบายธนาคาร) |
| **QR PromptPay / COD** | โอนเงินคืนเข้าบัญชีธนาคารตามที่ลูกค้าแจ้ง | 3-5 วันทำการ หลังอนุมัติ |

## **Part 3: ข้อกำหนดอ้างอิงสำหรับทีมกฎหมายและปฏิบัติการ (Internal Reference)**

### **1\. สถานะทางกฎหมายของแพลตฟอร์ม**

* BuyDi ทำหน้าที่เป็น **"ตลาดกลางทางพาณิชย์อิเล็กทรอนิกส์" (E-Marketplace)** ตามข้อกำหนด 4.2 ของ PRD
* แพลตฟอร์มทำหน้าที่เป็นตัวกลางในการถือครองเงิน (Escrow) จนกว่าธุรกรรมจะเสร็จสิ้นตามเงื่อนไขที่กำหนด

### **2\. การจัดการ Wallet (Financial Compliance)**

* **ผู้ที่มีสิทธิใช้ Wallet:** จำกัดเฉพาะผู้ขาย (Vendor) และครีเอเตอร์ (Affiliate) เพื่อใช้รับรายได้เท่านั้น
* **ข้อกำหนดความปลอดภัย:** ทุกการถอนเงิน (Withdrawal) จาก Wallet จะต้องผ่านการตรวจสอบจาก Admin และเป็นไปตามรอบการจ่ายเงินที่ระบุในนโยบายบัญชี

### **3\. การระงับข้อพิพาท (Dispute Resolution)**

* Admin มีอำนาจเด็ดขาดในการระงับยอดเงิน (Hold Payment) ใน Vendor Wallet หากมีการร้องเรียนจากผู้ซื้อภายในระยะเวลารับประกันตรวจสอบสินค้า 3 วัน จนกว่าข้อพิพาทจะสิ้นสุด

หมายเหตุ: ข้อมูลในเอกสารนี้สอดคล้องกับ PRD BuyDi 2026-02-11 หากมีการแก้ไขฟีเจอร์ในอนาคต เช่น การเพิ่มระบบสะสมคะแนนหรือ Wallet ผู้ซื้อ จะต้องมีการปรับปรุงเอกสารฉบับนี้ใหม่

# **คู่มือปฏิบัติการระบบ BuyDi**  **(Operational Manuals)**

*อ้างอิง PRD เวอร์ชัน 2.0 (2026-03-30) และนโยบายการเงิน 2025\_12\_19*

## **1\. คู่มือสำหรับผู้ขาย (Seller Operations Manual)**

**เป้าหมาย:** จัดการร้านค้า สินค้า และรายได้สุทธิ (Net Revenue) อย่างแม่นยำ

### **1.1 การลงทะเบียนและตรวจสอบตัวตน (Onboarding & KYC \- PRD 4.3.1)**

* **ประเภทบุคคลธรรมดา:** ต้องอัปโหลดรูปถ่ายบัตรประชาชนที่ชัดเจน และรูปถ่ายคู่กับบัตร (Selfie)
* **ประเภทนิติบุคคล:** ต้องใช้หนังสือรับรองบริษัท (อายุไม่เกิน 6 เดือน), ภพ.20 (ถ้ามี) และหน้าสมุดบัญชีในนามบริษัท
* **การผูกบัญชี:** ชื่อบัญชีธนาคาร "ต้องตรง" กับชื่อผู้สมัครหรือชื่อบริษัทเท่านั้น เพื่อป้องกันปัญหาการโอนเงินคืน (Refund) และการถอนเงิน

### **1.2 การจัดการสินค้าและสต็อก (Product & Stock \- PRD 4.3.5)**

* **Data Input:** ระบุชื่อสินค้าที่สื่อสารถึงแบรนด์ไทย, รายละเอียดเชิงลึก, และคำแนะนำการใช้งาน
* **Advanced Variations:** จัดการตัวเลือกสินค้า (เช่น ขนาด, สี, กลิ่น) โดยแต่ละรายการต้องมี SKU และจำนวนสต็อกแยกกันอย่างชัดเจน
* **Commission Setting (PRD 4.3.7):** ผู้ขายสามารถกำหนดค่าคอมมิชชันแยกตามรายสินค้าได้ (เช่น สินค้า A ให้ 5%, สินค้า B ให้ 10%) เพื่อดึงดูดครีเอเตอร์เฉพาะกลุ่ม

### **1.3 การจัดการคำสั่งซื้อและขนส่ง (Fulfillment)**

* **การจัดการคำขอคืนสินค้า:** เมื่อ Buyer ยื่นคำร้องขอคืนสินค้า ผู้ขายมีเวลา **3 วัน** ในการตอบกลับ หากไม่ตอบภายในกำหนด ระบบจะเลือกทางเลือก A (คืนสินค้า + คืนเงิน) อัตโนมัติ
* **สถานะการชำระเงิน:** ดำเนินการแพ็กสินค้าเฉพาะออเดอร์ที่มีสถานะ "Ready to Ship" เท่านั้น
* **Shipping Label:** ระบบเชื่อมต่อ GetShip/ไปรษณีย์ไทย ผู้ขายต้องพิมพ์ใบปะหน้าจากระบบเพื่อใช้ Tracking Number ที่ถูกต้อง
* **SLA การจัดส่ง:** ควรนำส่งสินค้าให้บริษัทขนส่งภายใน 24-48 ชม. เพื่อคะแนนร้านค้าที่ดี

### **1.4 ระบบการเงิน (Financial Flow \- PRD 4.3.8)**

* **กระบวนการ Escrow:** เงินค่าสินค้าจะถูกถือไว้โดยระบบเป็นเวลา **3 วัน** หลังส่งสำเร็จ (Delivered)
* **การคำนวณรายได้:**
  * ฐานคำนวณ = ยอดหลังหักส่วนลดร้านค้า (ไม่รวมค่าส่ง / ส่วนลดแพลตฟอร์ม)
  * หัก: Platform Fee % (ฐาน = ยอดหลังหักส่วนลดร้าน) + ค่าธุรกรรมการชำระเงิน % (ฐาน = ยอดที่ลูกค้าชำระ) + Affiliate Commission % (ฐาน = ยอดหลังหักส่วนลดร้าน) + ส่วนต่างค่าส่ง (ถ้า Actual > Estimated)
  * = **รายได้สุทธิร้านค้า**
* **กรณี Cancel:** ค่าธุรกรรมการชำระเงิน Platform รับภาระแทน ไม่เรียกเก็บจาก Seller (เรียกเก็บเฉพาะ Order = Complete)
* **การถอนเงิน:** ยอดเงินจะย้ายจาก "Pending" เป็น "Available" เมื่อจบระยะเวลา 3 วัน ผู้ขายกดถอนเงินได้ทันที (เงินเข้าบัญชีภายใน 3 วันทำการ)

### **1.5 ระบบจัดส่ง Non-COD และตรวจน้ำหนัก (Shipping Logic & Weight Check)**

* **ขั้นตอน:** Seller ยืนยันจัดส่ง → ขนส่งรับพัสดุ → ตรวจน้ำหนัก
* **Decision Tree น้ำหนัก:**

| กรณี | ผลลัพธ์ |
| ----- | ----- |
| **น้ำหนักตรง** (Actual = Estimated) | ออก RS Document → ส่งปกติ |
| **Actual > Estimated** | Seller จ่ายส่วนต่าง → ออก RSA Document |
| **Actual < Estimated** | ส่วนต่าง = BuyDi Revenue |

* **กรณีส่งไม่สำเร็จ / ตีกลับ:** คืนสินค้าให้ Seller → ยกเลิก/คืนเงิน Buyer → ออก RST + RS VOID

### **1.6 ระบบเก็บเงินปลายทาง COD (COD Shipping & Reconciliation)**

* **ขั้นตอน:** COD Order → จัดส่ง → Buyer จ่ายเงิน COD → ขนส่งเก็บเงิน → ปณ.โอน COD ให้ BuyDi → Generate Transaction ID + Validate → Update Status → Waiting COD Settlement → Order Completed → หักค่าขนส่ง → Available for Withdrawal
* **ความเสี่ยง COD:**

| ความเสี่ยง | รายละเอียด |
| ----- | ----- |
| **ปฏิเสธรับพัสดุ** | ส่งคืน Seller — ค่าส่งไป-กลับหัก Seller |
| **Remittance Delay** | ขนส่งโอนช้า 3-7 วัน ระบบต้อง track |
| **ยอดไม่ตรง** | Auto-flag + Manual review ถ้ายอดไม่ match |
| **ยกเลิก COD** | ไม่มี Refund (ยังไม่จ่ายเงิน) |

## **2\. คู่มือสำหรับครีเอเตอร์ (Affiliate Operations Manual)**

**เป้าหมาย:** สร้างรายได้จากการบอกต่อด้วยเครื่องมือ Tracking ที่แม่นยำ

### **2.1 การสมัครและการจัดการสิทธิ์ (Affiliate Setup \- PRD 4.4.1)**

* **Approval:** ต้องผ่านการอนุมัติสิทธิ์จาก Admin ก่อน จึงจะปรากฏเมนู Affiliate Dashboard
* **Link Generation (PRD 4.4.4):** ระบบใช้ระบบ Ref ID และ Cookie Tracking (**Cookie Window 30 วัน**) เพื่อบันทึกยอดขายแม้ลูกค้าจะไม่ได้ซื้อทันทีที่คลิก
* **Attribution Model:** ใช้ระบบ **Last Click** — คลิกสุดท้ายก่อนซื้อได้ Commission ทั้งหมด

### **2.2 เครื่องมือส่งเสริมการขาย (Promotion Tools \- PRD 4.4.7)**

* **Custom Coupons:** ครีเอเตอร์สามารถขอรหัสส่วนลดพิเศษ (ถ้ามีแคมเปญ) เพื่อนำไปแจกให้ผู้ติดตาม
* **Content Assets:** ดาวน์โหลดรูปภาพและวิดีโอรีวิวที่ผู้ขายอนุญาตจากหน้า Product Detail มาใช้งานได้

### **2.3 การติดตามรายได้และการจ่ายเงิน (Payout \- PRD 4.4.6)**

* **Real-time Tracking:** ดูยอด Click, ยอด Order และยอด Commission โดยประมาณการได้ทันที
* **Commission Lifecycle:** Create Order → Order Completed → อนุมัติ → ได้ Balance → ถอนเงิน (กรณี Order Cancelled → ไม่อนุมัติ Commission)
* **รอบการจ่ายเงิน:** ระบบจะทำการโอนค่าคอมมิชชันเข้าบัญชีธนาคารโดยอัตโนมัติ ตามรอบที่บริษัทกำหนด (เช่น ทุกวันที่ 15 และ 30 ของเดือน) หลังออเดอร์เสร็จสมบูรณ์

## **3\. คู่มือสำหรับผู้ดูแลระบบ (Admin Operations Manual)**

**เป้าหมาย:** ควบคุมธรรมาภิบาลของแพลตฟอร์มและการไหลเวียนทางการเงิน

### **3.1 การตรวจสอบคุณสมบัติ (Governance \- PRD 4.3.2)**

* **KYC Verification (SLA 24-48 ชั่วโมงทำการ):** ตรวจสอบความถูกต้องของบัตรประชาชน/หนังสือรับรอง และความชัดเจนของหน้าสมุดบัญชี
* **Product Moderation (SLA 24-48 ชั่วโมงทำการ):** ตรวจสอบรูปภาพสินค้า (ต้องไม่มีลายน้ำคู่แข่ง/ไม่เป็นสินค้าต้องห้าม) และตรวจสอบโครงสร้างราคากับค่าคอมมิชชัน
* **Affiliate Verification:** ตรวจสอบข้อมูล Social Media / บัญชีธนาคารของผู้สมัคร Affiliate และอนุมัติ/ปฏิเสธ

### **3.2 การจัดการการเงินและบัญชี (Financial Management \- PRD 4.3.8)**

* **อนุมัติการถอนเงิน (Withdrawal Approval):** 1\. ตรวจสอบว่าออเดอร์ที่เกี่ยวข้องพ้นระยะ 3 วันจริงหรือไม่ 2\. ยืนยันยอดเงินที่หัก GP และค่าธรรมเนียมขนส่ง 3\. กดอนุมัติเพื่อให้ระบบโอนเงินผ่าน Mass Pay API
* **การออกเอกสาร:** ระบบจะ Generate เอกสารทางการเงินอัตโนมัติตาม Event (ดูหัวข้อ 3.5)

### **3.3 การจัดการข้อพิพาทและคืนเงิน (Dispute & Refund)**

* **Escrow Hold:** หากมีการแจ้ง "คืนสินค้า" Admin ต้องกด Hold เงินในออเดอร์นั้นทันทีในระบบหลังบ้าน
* **Investigation:** เรียกดูหลักฐานวิดีโอการแกะกล่องจากฝั่งผู้ซื้อ และหลักฐานการแพ็กจากฝั่งผู้ขาย
* **Refund Execution:** หากตัดสินให้คืนเงิน Admin ต้องทำการ Refund กลับสู่ช่องทางชำระเงินเดิมของลูกค้า (Void Credit Card / Bank Transfer)

### **3.4 การจัดการอัตราค่าคอมมิชชัน (Commission Rate Management)**

* **การกำหนดอัตรา:** Admin สามารถตั้งค่าและเปลี่ยนแปลงอัตราค่าคอมมิชชันแพลตฟอร์ม (Platform Fee) ตามหมวดหมู่สินค้าได้ผ่านระบบหลังบ้าน
* **ผลบังคับใช้:** อัตราใหม่จะใช้กับคำสั่งซื้อใหม่เท่านั้น ไม่มีผลย้อนหลัง
* **ความโปร่งใส:** แสดงอัตราค่าคอมมิชชันอย่างชัดเจนให้ผู้ขายทราบก่อนลงรายการสินค้า

### **3.5 การจัดการแคมเปญ (Marketing)**

* **Flash Sales:** ตั้งค่าช่วงเวลาและสินค้าที่เข้าร่วมกิจกรรม พร้อมจำกัดจำนวนสต็อกพิเศษ
* **Platform Coupons:** สร้างรหัสส่วนลดที่แพลตฟอร์มเป็นผู้สนับสนุนค่าใช้จ่ายเอง

### **3.6 เอกสารทางการเงิน 6 ประเภท (Financial Documents)**

แพลตฟอร์มออกเอกสารอัตโนมัติตาม Event — Marketplace ไม่ออกเอกสารขายแทน Seller

**เมื่อ Order = Pending:**

| เอกสาร | Trigger | ออกให้ | หมายเหตุ |
| ----- | ----- | ----- | ----- |
| **PV** — Platform Discount Voucher | Buyer ใช้ส่วนลดแพลตฟอร์ม | Seller (เก็บใน Admin) | ยกเลิก Order → VOID |
| **RS** — Receipt Shipping | Payment Completed | Internal/Admin | ยกเลิก/คืนสินค้า → VOID |

**เมื่อ Order = Complete:**

| เอกสาร | Trigger | ออกให้ | หมายเหตุ |
| ----- | ----- | ----- | ----- |
| **RV** — GP Invoice/Receipt | Order Complete (หัก GP + ค่าธุรกรรม) | Seller (Seller ปริ้นได้) | ยกเลิกหลัง Complete → VOID |
| **RSA** — Shipping Adjustment | Actual Weight > Estimated | Seller (Seller ปริ้นได้) | ไม่มีส่วนต่างเพิ่ม = ไม่ออก |

**เมื่อ Withdraw / Refund:**

| เอกสาร | Trigger | ออกให้ | หมายเหตุ |
| ----- | ----- | ----- | ----- |
| **PVW** — Payment Voucher Withdraw | โอนเงินจริงสำเร็จ | Seller/Affiliate (เก็บใน Admin) | — |
| **RST** — Return Shipping Receipt | Refund Completed / ส่งไม่สำเร็จ-ตีกลับ | Seller (Seller ปริ้นได้) | คืนเงินอย่างเดียว → ค่าส่งขาไป / คืนสินค้า → 2 ขา |

**ข้อจำกัด:** แก้ไขตัวเลขไม่ได้ / Seller เห็นเฉพาะของตน / Trace กลับ Event ได้ / Void = Watermark แต่ยังดูได้

### **3.7 ระบบถอนเงินและจ่ายเงิน (Withdrawal & Payout Flow)**

**แหล่งที่มาของ Withdrawal:**

| แหล่งที่มา | ตัวอย่าง | การอนุมัติ |
| ----- | ----- | ----- |
| **Wallet Withdrawal** | Affiliate ถอน Commission / Seller ถอนจาก Wallet | Manual → Admin Review |
| **Buyer Triggers** | Cancel (จ่ายแล้ว) / Return success / Dispute ชนะ | Auto Approved |
| **Seller Triggers** | Seller cancel (buyer จ่ายแล้ว) / Accept return / Dispute Buyer wins | Auto Approved |

**Admin Payout Process:**
1. Approved → Create Batch → Export to Bank → Bank โอนเงิน → Close Batch
2. Transfer OK → Completed + ออก Payment Voucher (PVW)
3. Transfer Failed → กลับเป็น Approved รอรอบถัดไป

**Admin Wallet Withdrawal Review:**
* Pending Approval → Admin Reviews → Approve หรือ Reject (คืน Wallet)
