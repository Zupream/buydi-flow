05 / 17
Return, Refund &
Dispute Resolution
Precondition: Order = Delivered + ภายใน 3 วัน + Buyer แนบหลักฐาน
🙋 Buyer ยื่นคำร้อง
→
🏪 Seller ตรวจคำขอ (3 วัน)
SELLER เลือก 3 ทาง (ไม่ตอบใน 3 วัน = Auto เลือก A)
A: คืนสินค้า + คืนเงิน
  • Buyer ส่งคืนภายใน 3 วัน
  • Seller ตรวจของ (3 วัน ไม่ตรวจ = auto approve)
  • Seller รับผิดชอบค่าส่ง 2 ขา
  • Admin คืนเงิน Buyer
B: คืนเงินอย่างเดียว
  • ไม่ต้องคืนสินค้า
  • ข้าม process ขนส่ง
  • Seller สละสิทธิ์รับสินค้าคืน
  • Admin คืนเงิน Buyer ทันที
C: ไม่อนุมัติ
  • Seller ใส่เหตุผล
  • Buyer โต้แย้งได้ใน 3 วัน
  • ไม่โต้แย้ง = ปิดเคส (Order Complete)
  • โต้แย้ง → Admin ตัดสิน
DISPUTE — ADMIN ตัดสิน (ผลถือเป็นที่สิ้นสุด)
✅ Buyer Win
  • สินค้าอยู่ที่ Buyer → คืนเงินเต็ม + เก็บสินค้า
  • สินค้าอยู่ที่ Seller → คืนเงินเต็ม + Seller รับค่าส่ง 2 ขา
❌ Seller Win
  • สินค้าอยู่ที่ Buyer → ไม่คืนเงิน + Buyer เก็บสินค้า
  • สินค้าอยู่ที่ Seller → ไม่คืนเงิน + Seller รับค่าส่ง 1 ขา (ขากลับ)

  • ค่าธุรกรรมการชำระเงิน % (ฐาน = ยอดที่ลูกค้าชำระ)
  • Affiliate Commission % (ฐาน = ยอดหลังหักส่วนลดร้าน)
  • ส่วนต่างค่าส่ง (ถ้า Actual > Estimated)
📊 สูตรคำนวณ
  • ยอดหลังหักส่วนลดร้านค้า

  • − ค่าธุรกรรมการชำระเงิน %
  • − ส่วนต่างค่าส่ง (ถ้ามี)
  • = รายได้สุทธิร้านค้า
⚠️ กรณี Cancel

  • ไม่เรียกเก็บจาก Seller
  • เรียกเก็บเฉพาะ Order = Complete

Shipping Cost
Adjustment Decision Tree
ตัดสินใจเรื่องค่าส่ง — GetShip Webhook ได้น้ำหนักจริง
1️⃣ น้ำหนักตรง
  • RS Document
  • ไม่ต้องปรับ
2️⃣ Actual > Estimated
  • RS Document + RSA Document
  • หักจาก Seller Wallet
  • แจ้ง Seller
3️⃣ Actual < Estimated
  • RS Document
  • ส่วนต่าง = BuyDi Revenue
  • ไม่ต้องแจ้ง Seller
RST — ค่าจัดส่งจากการคืนสินค้า / ส่งไม่สำเร็จ
📄 RST กรณีคืนเงินอย่างเดียว
  • ค่าส่งขาไป
  • ออกให้ Seller
📄 RST กรณีคืนเงิน + คืนสินค้า
  • ค่าส่งขาไป + ขากลับ
  • ออกให้ Seller
📄 RST กรณีส่งไม่สำเร็จ/ตีกลับ
  • ค่าส่งขาไป (+ RSA ถ้ามีส่วนต่าง)
  • ออกให้ Seller

Financial Documents
6 ประเภทเอกสาร
เอกสารทางการเงินที่ Platform ออกอัตโนมัติตาม Event — Marketplace ไม่ออกเอกสารขายแทน Seller
ORDER = PENDING
📄 PV — Platform Discount Voucher
  • ออกเมื่อ Buyer ใช้ส่วนลดแพลตฟอร์ม
  • ออกให้ Seller (เก็บใน Admin)
  • ยกเลิก Order → VOID
📄 RS — Receipt Shipping
  • ออกเมื่อ Payment Completed
  • เก็บ Internal/Admin (ไม่ส่ง Buyer อัตโนมัติ)
  • ยกเลิก/คืนสินค้า → VOID
ORDER = COMPLETE
📄 RV — GP Invoice/Receipt
  • หัก GP + ค่าธุรกรรมการชำระเงิน
  • ออกให้ Seller (Seller ปริ้นได้)
  • ยกเลิกหลัง Complete → VOID
📄 RSA — Shipping Adjustment
  • ออกเมื่อ Actual > Estimated (ส่วนต่างค่าส่ง)
  • ออกให้ Seller (Seller ปริ้นได้)
  • ไม่มีส่วนต่างเพิ่ม = ไม่ออก
WITHDRAW / REFUND
📄 PVW — Payment Voucher Withdraw
  • ออกเมื่อโอนเงินจริงสำเร็จ
  • ออกให้ Seller/Affiliate (เก็บใน Admin)
  • Affiliate: ไม่แสดงภาษีหัก ณ ที่จ่าย
📄 RST — Return Shipping Receipt
  • ออกเมื่อ Refund Completed / ส่งไม่สำเร็จ-ตีกลับ
  • ออกให้ Seller (Seller ปริ้นได้)
  • คืนเงินอย่างเดียว → ค่าส่งขาไป / คืนสินค้า → 2 ขา
